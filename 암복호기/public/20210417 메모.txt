DRY 원칙
Don't Repeat Yours 
같은 것을 반복하지 말라- 일반화
일반적인 내용이 같고 특정 부분만 다른 코드가 두개(혹은 그 이상) 있을 때 
같은 부분에 다른 부분을 빼고 밖에서 여러 코드를 불러오는 방법

ex)
22222 코드1 33333
22222 코드2 33333

22222 (비워둠) 33333

코드1, 코드2 를 밖에 두고 22222와 33333 사이에 필요한 코드를 넣어준다.


일반화가 아닌 if문을 활용한 방법
if문의 ~~라면 을 사용해 ㅇㅇ하다면, 만약에 아니라면 으로 여러 코드를 넣어두고 각각 특정한 상황이 지정된 코드가 실행되게 해준다.
단, if문을 사용하면 각각 특정한 상황에서만 반응하기 때문에 특별하다.

ex) if 2222하다면 실행할 코드
   2222하지 않고 333한다면 실행할 코드(아니라면)
   2222하지 않고 444한다면 실행할 코드(아니라면)
   2222하지 않고 555한다면 실행할 코드(아니라면)


INLINE_인라인 함수
보통 밖에 function을 정의하고 이름을 붙여서 여러 곳에서 함수를 호출하지만 한 곳에서만 호출하는 함수에는 굳이 따로 빼서 이름을 붙일 이유가 없다.
그래서 새로 만들지 않고 변수에 바로 함수를 넣어주는 방법을 사용하기도 한다. 이 때 함수의 이름은 지정하지 않는다.(한 변수에 값을 바로 넣어버리기 때문)
하지만 인라인함수를 이용하면 각각 임무가 정확히 부여되어있던 함수들이 코드 하나로 합쳐져 있기 때문에 훨씬 복잡해서 한 곳에서만 사용하는 함수라고 무조건 인라인함수를 쓰는게 좋다고 할 수는 없다.

ex)
변수1 = 인라인함수() {
};

func1 = function() {
}
